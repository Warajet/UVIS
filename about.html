Flexible IoT Data Platform

About

Flexible IoT Data Platform for Geospatial Data with Visualisation

The objective of this project is to develop a server system that is flexible and accept geographical data from any IoT weather stations with low criteria that server needs to explicitly understand the data. The server system must also be able to use the collected geospatial data and export it for visualisation where the visualisation must adapt to the server system. These following goals must be achieved to indicate that the project is successful.


Scope

The flexible IoT data platform for geospatial data with visualisation has the scope lists below as follow:

  • To develop a server which are flexible to accept partially structured geospatial data from any sources without the need of an explicit knowledge of the data.
  • Collect the various type of data from various sources and join various data in meaningful format which is readable by human.
  • The flexible IoT platform must be able to at least receive temperature, air humidity, air pressure and wind speed.
  • The flexible IoT platform must be able to collect real time data at a minimum pace of daily rate.
  • The flexible IoT platform must be able to connect to at least 5 IoT sensors gather across Thailand.
  • The flexible IoT platform must be able to use geospatial data it receives for visualisation
  • The visualisation should be on web-based platform.
  • The visualisation must be able to display the location of each IoT platform on the world map.
  • The visualisation must be able to precisely and clearly display a map with data overlay and supports importing various geospatial data sources as layers.
  • The visualisation data overlay should includes rain, wind, PM2.0 and traffic.
  • The visualisation can adapt to qualitative and quantitative changes of weather stations.
